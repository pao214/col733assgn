  3  ConcurrentRotatingFileHandler:  This class is a log handler which is a drop-in 
  4  replacement for the python standard log handler 'RotateFileHandler', the primary 
  5  difference being that this handler will continue to write to the same file if 
  6  the file cannot be rotated for some reason, whereas the RotatingFileHandler will 
  7  strictly adhere to the maximum file size.  Unfortunately, if you are using the 
  8  RotatingFileHandler on Windows, you will find that once an attempted rotation 
  9  fails, all subsequent log messages are dropped.  The other major advantage of 
 10  this module is that multiple processes can safely write to a single log file. 
 11   
 12  To put it another way:  This module's top priority is preserving your log 
 13  records, whereas the standard library attempts to limit disk usage, which can 
 14  potentially drop log messages. If you are trying to determine which module to 
 15  use, there are number of considerations: What is most important: strict disk 
 16  space usage or preservation of log messages? What OSes are you supporting? Can 
 17  you afford to have processes blocked by file locks? 
 18   
 19  Concurrent access is handled by using file locks, which should ensure that log 
 20  messages are not dropped or clobbered. This means that a file lock is acquired
